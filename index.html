VS Visible Security

보이지 않던 취약점과 공격받은 루트의 경로를 현장 시야 위에 겹쳐 보여주는 디지털 안경. VS는 장비, 네트워크, 출입 동선의 위험 신호를 사람이 바로 판단할 수 있는 시각 언어로 바꿉니다.

01 / Vulnerability Lens

기기 취약점을 눈앞에 표시

펌웨어 버전, 열린 포트, 미승인 연결처럼 눈으로 볼 수 없는 위험을 장비 위치 위에 바로 띄웁니다.

02 / Attack Route

공격받은 루트의 흐름 추적

최초 진입점부터 내부 이동, 권한 상승 지점까지 시간순 경로로 연결해 대응 우선순위를 줄입니다.

03 / Field Decision

현장에서 바로 판단

관제실의 복잡한 로그를 착용자의 시야에 맞춘 짧은 신호, 색상, 거리 정보로 압축합니다.

보안 로그가 공간 지도로 바뀌는 순간

VS는 네트워크 공격을 화면 속 표가 아니라 실제 공간 위의 선과 노드로 보여줍니다. 사용자는 복도, 서버랙, 회의실 단말기를 바라보며 어떤 취약점이 어떤 경로로 연결됐는지 즉시 파악합니다.

1
장비 인식안경이 시야 안의 라우터, 카메라, 키오스크, 서버랙을 분류하고 보안 상태를 겹쳐 표시합니다.
2
위험 점수화취약점의 심각도, 노출 범위, 최근 공격 흔적을 합쳐 색상과 거리감으로 표현합니다.
3
루트 재구성침입자가 이동한 흔적을 선으로 연결해 봉쇄할 장비와 확인할 계정을 좁힙니다.
VISOR MAPROUTE: ACTIVE

보안 담당자가 움직이는 곳마다 시야가 관제 화면이 됩니다.

공격을 발견한 뒤, VS는 다음 행동까지 보여줍니다.

고객은 단순히 “위험함”을 듣는 것이 아니라 어떤 장비를 격리하고, 어떤 계정을 잠그고, 어떤 증거를 남겨야 하는지 현장 순서대로 확인할 수 있습니다.

01

확인

VS가 공격 시작점, 이동 경로, 영향 장비를 시야 위에 표시해 오탐 여부와 우선순위를 빠르게 구분합니다.

02

격리

감염 의심 장비, 취약 AP, 비정상 계정을 분리 대상으로 표시하고 현장 담당자가 바로 봉쇄할 수 있게 안내합니다.

03

복구

패치, 비밀번호 초기화, 접근 정책 재적용처럼 필요한 조치를 장비별 체크리스트로 보여줍니다.

04

보고

공격 경로, 조치 시간, 남은 위험을 스냅샷으로 저장해 내부 보고와 재발 방지 회의에 활용합니다.

Example A / Critical

회의실 공유기가 침입 경로로 확인된 경우

VS 렌즈가 공유기에서 서버실 장비로 이어지는 비정상 연결을 붉은 경로로 표시합니다.

  • 공유기를 게스트망에서 즉시 분리하고 임시 차단 정책을 적용합니다.
  • 같은 경로에 연결된 계정의 세션을 종료하고 비밀번호를 초기화합니다.
  • 공격 경로 스냅샷을 저장해 보안팀과 시설팀이 같은 화면으로 대응합니다.
Example B / Warning

CCTV 펌웨어 취약점이 악용된 경우

취약 펌웨어가 설치된 카메라와 내부 관리 콘솔 사이의 이동 흔적을 노란 경로로 보여줍니다.

  • 해당 카메라를 격리망으로 이동하고 외부 접근을 임시 차단합니다.
  • 동일 모델 장비를 한 번에 찾아 패치 우선순위를 지정합니다.
  • 관리 콘솔 접속 기록을 보존하고 의심 계정의 권한을 낮춥니다.
Example C / Recovery

서버랙까지 접근 흔적이 남은 경우

공격자가 거쳐 간 AP, 중계 장비, 서버랙을 시간순으로 재구성해 복구 순서를 제안합니다.

  • 서버랙 연결을 핵심 서비스와 비핵심 서비스로 나눠 단계적으로 격리합니다.
  • 관리자 계정, API 키, 원격 접속 정책을 순서대로 재검증합니다.
  • 복구 완료 후 남은 취약점을 녹색 체크와 회색 보류 항목으로 분리합니다.

Visible Security turns invisible risk into a visible project story.

VS는 가상의 제품 콘셉트를 바탕으로 만든 프로젝트형 웹사이트입니다. 취약점 발견, 공격 경로 시각화, 대응 흐름까지 하나의 경험으로 보여주며 “보안이 보인다”는 아이디어를 분명하게 전달합니다.
